# Contributing to Linux DevOps Toolkit

Thank you for your interest in contributing! This document provides guidelines and information for contributors.

## How to Contribute

### Reporting Bugs
1. Check existing issues to avoid duplicates
2. Use the bug report template when creating an issue
3. Include: steps to reproduce, expected vs actual behavior, environment details (OS, Python version, etc.)

### Suggesting Features
1. Check if the feature has already been requested
2. Use the feature request template
3. Explain the use case and why it benefits users
4. Consider if it fits the scope of existing tools or warrants a new tool

### Submitting Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Add tests for new functionality
5. Ensure all tests pass (`make test` or appropriate command)
6. Commit with clear, descriptive message
7. Push to your fork (`git push origin feature/amazing-feature`)
8. Open a Pull Request with detailed description

### Code Style
- Follow existing code conventions
- Use meaningful variable/function names
- Keep functions small and focused (< 50 lines where possible)
- Document public APIs with docstrings or comments
- Use type hints in Python where helpful

### Testing
- Write unit tests for new code
- Ensure existing tests pass
- Test on Linux (primary target)
- Include integration tests if applicable
- For new tools, include at least basic test coverage

### Commit Messages
```
type(scope): brief description

Longer explanation if needed.

Closes #ISSUE_NUMBER
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `automated`

### Development Setup

```bash
# Clone and setup
git clone https://github.com/unn-Known1/linux-service-manager.git
cd linux-service-manager

# Install dependencies for a specific tool
cd tools/some-tool
make deps

# Run tests
make test

# Run the tool
make run
```

## Adding a New Tool

New tools should be added via the Gap Analyzer:

```bash
# Find gaps in the market
python3 gap-analyzer.py --analyze

# Expand an idea (creates scaffold in tools/)
python3 gap-analyzer.py --expand <IDEA_ID>

# Update main README
python3 update-readme.py
```

Alternatively, you can manually create a directory under `tools/`:
- Create `tools/<tool-name>/README.md`
- Create implementation file (`<tool-name>.py`, `.go`, `.js`, or `.sh`)
- Add tests in `tools/<tool-name>/tests/`
- Add a Makefile for convenience
- Update main README via `update-readme.py`
